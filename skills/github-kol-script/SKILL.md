# GitHub 爆款项目口播文案生成器

自动发现 GitHub 热门项目，评估爆款潜力，生成适合多平台的高转化口播脚本。

---

## 使用方式

### 模式一：自动发现爆款（推荐）
```
/github-kol-script --discover              # 自动发现今日爆款项目
/github-kol-script --discover --category ai      # 发现 AI 类爆款
/github-kol-script --discover --category devtools # 发现开发工具类
/github-kol-script --discover --platform 抖音    # 生成抖音专用文案
```

### 模式二：分析指定项目
```
/github-kol-script https://github.com/owner/repo
/github-kol-script owner/repo
/github-kol-script vercel/next.js --style 硬核技术
```

### 模式三：批量生成
```
/github-kol-script --batch 5              # 一次性生成 5 个爆款项目文案
/github-kol-script --trending weekly      # 分析本周 Trending
```

### 模式四：生成视频（Beta）
```
/github-kol-script --discover --generate-video    # 发现项目并自动生成视频
/github-kol-script microsoft/markitdown --generate-video  # 为指定项目生成视频
```
**视频生成功能说明**：
- 基于 Remotion 自动生成 60 秒抖音竖屏视频
- 使用 ~/demo/markitdown-video 项目模板
- 自动替换项目信息、录屏演示、字幕文案
- 输出 1080x1920 60fps MP4 格式

## 爆款项目识别系统

### 1. 爆款项目来源渠道

执行 `--discover` 时，按优先级从以下渠道获取：

| 优先级 | 来源 | 获取方式 | 更新频率 |
|--------|------|----------|----------|
| 1 | GitHub Trending | https://github.com/trending | 实时 |
| 2 | Hacker News | https://news.ycombinator.com/show | 每小时 |
| 3 | Product Hunt | https://www.producthunt.com | 每日 |
| 4 | 大厂开源动态 | vercel/meta/google/microsoft 等 | 实时 |
| 5 | 技术社区热帖 | Reddit r/programming, lobste.rs | 每日 |

### 2. 爆款潜力评分模型

每个项目按以下维度评分（满分 100）：

```
爆款潜力分 = 基础分(40) + 内容分(35) + 时机分(25)
```

#### 基础分（项目本身属性）

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| Stars 增速 | 15 | 今日新增 >500: 15分；100-500: 10分；<100: 5分 |
| 项目热度 | 10 | Stars >1万: 10分；1k-1万: 7分；<1k: 5分 |
| 活跃度 | 10 | 最近1周有提交: 10分；1月内: 7分；>1月: 3分 |
| 易用性 | 5 | 一键安装/开箱即用: 5分；需配置: 3分；复杂部署: 1分 |

#### 内容分（口播呈现潜力）

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| 视觉可展示 | 12 | 有界面/效果可录屏: 12分；终端展示: 8分；纯代码: 3分 |
| 痛点明确度 | 10 | 解决普遍痛点: 10分；特定场景: 6分；小众需求: 3分 |
| 情绪价值 | 8 | "用了回不去"/惊艳感: 8分；好用: 5分；一般: 2分 |
| 故事性 | 5 | 有大厂背书/创始人故事: 5分；社区驱动: 3分 |

#### 时机分（流量窗口）

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| 话题热度 | 15 | AI/热门技术栈: 15分；常青工具: 10分；过时技术: 2分 |
| 新鲜度 | 10 | 本周发布/重大更新: 10分；本月: 7分；半年内: 4分 |

### 3. 爆款项目分类标签

自动识别项目类型，匹配最佳内容策略：

| 分类标签 | 定义 | 爆款公式 | 适合平台 |
|----------|------|----------|----------|
| **AI编程工具** | AI辅助写代码 | "AI自动生成+效率翻倍" | 抖音/B站 |
| **终端神器** | 终端/Shell工具 | "程序员必备+装逼神器" | 抖音/小红书 |
| **效率工具** | 开发效率提升 | "省时间+告别繁琐" | 小红书/视频号 |
| **替代方案** | 商业软件开源替代 | "免费替代+放弃XX" | 抖音/B站 |
| **新版本发布** | 热门框架更新 | "重大更新+5个新功能" | 全平台 |
| **视觉工具** | UI/可视化/图表 | "效果惊艳+前后对比" | 小红书/抖音 |
| **学习资源** | 教程/路线图/资料 | "自学神器+少走弯路" | 小红书/B站 |
| **基础设施** | 后端/运维/DevOps | "大厂都在用+性能炸裂" | B站/视频号 |

## 执行流程

### 模式一：自动发现爆款流程

```
用户输入: /github-kol-script --discover

步骤 1: 获取数据源
  ├── 获取 GitHub Trending (今日)
  ├── 获取 Hacker News 热榜
  ├── 获取 Product Hunt 今日产品
  └── 合并去重，得到候选列表

步骤 2: 爆款潜力评分
  ├── 对每个候选项目计算「爆款潜力分」
  ├── 按分数降序排序
  └── 筛选分数 >70 的项目

步骤 3: 爆款项目展示
  └── 输出 TOP 5 项目列表：
      | 排名 | 项目 | 分类 | 潜力分 | Stars增速 |
      | 1 | xxx | AI工具 | 92 | +1.2k |
      | 2 | xxx | 效率工具 | 88 | +800 |
      ...

步骤 4: 生成口播文案
  └── 对 TOP 1 项目生成全平台文案

步骤 5: 输出优化建议
  └── 标题、封面、标签、发布时间建议

步骤 6: 保存报告到本地
  └── 生成 Markdown 文件
  └── 保存路径: ~/demo/hot/
  └── 文件名: github-hot-YYYYMMDD-HHMMSS.md
  └── 输出保存成功提示
```

### 模式二：指定项目分析流程

```
步骤 1: 获取项目信息
  ├── 解析 owner/repo
  ├── 获取 GitHub API 数据
  └── 分析 README 内容

步骤 2: 爆款潜力评估
  └── 计算该项目在「爆款评分模型」中的得分

步骤 3: 分类标签识别
  └── 自动识别项目类型（AI工具/终端神器/效率工具...）

步骤 4: 匹配最佳文案模板
  └── 根据项目类型选择对应的爆款文案结构

步骤 5: 生成多平台文案
  └── 抖音/小红书/B站/视频号 四个版本

步骤 6: 输出优化建议
```

## 爆款文案质量检查清单

### 基础检查（所有平台）

- [ ] 开头 3 秒有强钩子（疑问/震惊/反差）
- [ ] 明确指出了用户痛点（你是不是也...）
- [ ] 解决方案描述清晰（一句话说清是什么）
- [ ] 有具体数据/效果支撑（省多少时间/性能提升多少）
- [ ] 有行动号召（关注/收藏/试用/评论）
- [ ] 项目链接正确无误

### 平台专项检查

**抖音/快手：**
- [ ] 节奏快，信息密度高
- [ ] 有情绪起伏（惊讶→好奇→兴奋）
- [ ] 有视觉展示点（可录屏演示）
- [ ] 时长控制在 45-60 秒

**小红书：**
- [ ] 有 emoji 点缀，排版美观
- [ ] 有数字列表（1️⃣2️⃣3️⃣）
- [ ] 有个人体验/使用感受
- [ ] 有 3-5 个相关话题标签

**B站/YouTube：**
- [ ] 有清晰的分段结构
- [ ] 有技术原理讲解
- [ ] 有代码/效果演示
- [ ] 有与其他方案的对比

**视频号：**
- [ ] 语气专业可信
- [ ] 有真实案例/大厂背书
- [ ] 有价值总结/方法论

## 🎬 视频生成功能（v3.0 Beta）

### 功能概述

自动基于分析结果生成 60 秒抖音竖屏视频，使用 Remotion 框架渲染。

**视频特点**：
- 抖音竖屏 9:16 格式 (1080x1920)
- 60 秒时长，30fps
- 6 场景结构：钩子 → 痛点 → 方案 → 演示 → 爆点 → CTA
- 智能缩放动画，清晰展示重点内容
- 自动适配项目信息

### 使用方法

```bash
# 发现项目并生成视频
/github-kol-script --discover --generate-video

# 为指定项目生成视频
/github-kol-script owner/repo --generate-video

# 生成视频并指定输出目录
/github-kol-script owner/repo --generate-video --output ~/videos/
```

### 视频生成流程

```
步骤 1: 分析项目
  └── 获取项目信息、评分、文案

步骤 2: 准备视频模板
  └── 使用 ~/demo/markitdown-video 项目
  └── 复制模板到临时目录

步骤 3: 动态替换内容
  └── 替换 HookScene: 项目钩子文案
  └── 替换 PainPointScene: 项目痛点描述
  └── 替换 SolutionScene: 项目名称 + Logo
  └── 替换 DemoScene: 命令示例 + 输出预览
  └── 替换 HighlightScene: 核心卖点数据
  └── 替换 CTAScene: GitHub 链接 + Stars 数

步骤 4: 渲染视频
  └── 执行 npm run build
  └── 生成 MP4 文件

步骤 5: 输出结果
  └── 视频路径: ~/demo/videos/[owner]-[repo]-YYYY-MM-DD.mp4
  └── 文件大小、时长信息
```

### 视频模板说明

**模板位置**: `~/demo/markitdown-video/`

**模板结构**:
```
markitdown-video/
├── src/
│   ├── components/
│   │   ├── Zoomable.tsx      # 缩放动画组件
│   │   ├── Subtitle.tsx      # 字幕组件
│   │   └── AnimatedText.tsx  # 文字动画
│   ├── compositions/         # 6个场景组件
│   ├── Root.tsx              # 场景组合
│   └── index.ts
├── package.json
└── remotion.config.ts
```

**可替换内容**:
| 场景 | 可替换元素 | 示例 |
|------|-----------|------|
| HookScene | 问题文案 | "你是不是也遇到过..." |
